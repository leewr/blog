viepports解析

發表於 2016-01-17

在移动端中你或许已经使用过这样的meta声明，但你不一定清楚所有的细节。下面将在这文中向你讲述meta相关的内容。

概念：像素(pixel)

我们所说的像素实际上有设备像素、css像素两种。

  1. 设备像素：设备像素的物理像素，任何设备的物理像素的数量是固定的。
  2. css像素：为web开发者创造的，在css和javascript中抽象的层。
    在大多数情况下我们能够利用 screen.width/height,获得具体的设备pixels。如我们在开发中设置一个dix的宽度为200px，当我们对浏览器进行缩放，如200%，此是div的宽度仍然是200px，缩放并没有应为200%而变成了100px。尽管此时它占用了400px设备像素。
    换据话说，缩放200%一个单位的css像素变成了4倍的pixels。即宽度2高度2。

100%缩放

在通常情况下我们会在100%缩放条件下测试你的网站，即便用户缩放，css的pixels仍然能够保证你网站的外观比例。

屏幕尺寸 screen size

  • 屏幕的尺寸即屏幕的完整大小。
  • 设备的pixels
  • 兼容性问题：ie8、9浏览器中处于缩放状态得到的为缩放后屏幕大小的css尺寸

浏览器尺寸 window size

  • window.innerWidht/Height获得的浏览器完整尺寸
  • 兼容性IE不支持，opera用设备pixels来度量。

我们通过改变浏览器的窗口大小，这些数量将会随之而改变。当我们缩放页面尺寸时如200%，此时我们将获得的浏览器窗口尺寸将会减小。
opera却获得的值会变大。是一个放大后的css像素值。

需要注意的是，窗口内部高度和宽度的尺寸包含了滚动条的尺寸。

滚动位移 scroling offset

window.pageXOffset/pageYOffset

  • 页面的位移
  • 单位：csspixels
  • 兼容性问题：pageXOffset和pageYOffset在IE8之前的版本不支持，使用document.body.sceollLeft/scrollTop来取代。

视窗 viewport

viewport的功能在于控制你的网站块容器元素宽度。
当你改变浏览器窗口的大小时候，html元素的大小也随之改变。在桌面浏览上，viewport严格等于你的浏览器大小。

viewport的值 document.documentElement.clientWidht/Height

我们知道document.documentElement实际上就是html元素，html文档的根元素。而viewport是容纳html元素的元素。他是有浏览器的决定的。怎么说，当我们设置html元素25%时，document.documentElement.clientWidth/Height只会给出viewport的尺寸。
你可以看这个demo

那当html宽度大于浏览器的宽度时候，此时的viewport是多少呢？
我们设置html 1000px;浏览器窗口小于100时候查看document.documentElement.clientWidht的值可以发现是浏览器的窗口大小值。
我们需要记住的就是viewport等于你的浏览器尺寸，不管你页面的内容大小。

当然我们要细追究下去就是 window.innerWidth/Height包含滚动条
document.documentElement.clientWidth/Height不包含滚动条

媒体查询 media queries

@media css属性，你可以根据页面的特定宽度定义特殊的css规则。例如：

div.container{width:1000px;}
@media all and(max-width:1000px){
    div.container{width:800px;}
}

如果宽度小于1000px;那么container的宽度为800px;此处的宽度是什么宽度呢？此处的宽度是浏览器的窗口大小。

移动端上浏览器的问题

简言之就是我们无法像pc端一样处理移动端的问题，因为设备宽度的问题。通常只有400px，或者更小。
